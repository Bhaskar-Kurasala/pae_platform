# Follow-ups

One file per outstanding action that came out of a PR but couldn't
be resolved inside it. Each file:

- Names a clear owner (or "unassigned — please claim")
- Has a "Done when" checklist
- Is committed alongside the PR that created it, so the obligation
  travels with the code instead of evaporating in a chat thread

Resolved follow-ups stay in the directory with a "Resolved
YYYY-MM-DD by @user" line on top. The trail of past follow-ups is
itself useful — it tells the next operator what kind of pre-deploy
verifications this codebase has needed before.
