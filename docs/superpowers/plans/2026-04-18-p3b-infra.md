## Task 2: #159 Request ID propagation

**Critic check:** When a request fails, there's no way to correlate frontend error with backend log. A request ID threads the two together.

**Files:**

- Modify: `backend/app/main.py`

- [ ] **Step 1: Claim #159**

- [ ] **Step 2: Create request_id.py middleware**

```python
# backend/app/core/request_id.py
import uuid
from typing import Any
import structlog
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

REQUEST_ID_HEADER = "X-Request-ID"

class RequestIDMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next: Any) -> Response:
        request_id = request.headers.get(REQUEST_ID_HEADER) or str(uuid.uuid4())
        structlog.contextvars.clear_contextvars()
        structlog.contextvars.bind_contextvars(request_id=request_id)
        response = await call_next(request)
        response.headers[REQUEST_ID_HEADER] = request_id
        return response
```

- [ ] **Step 3: Register middleware in main.py**

# In backend/app/api/v1/routes/health.py — replace or extend:
from fastapi import APIRouter
from sqlalchemy import text
from sqlalchemy.ext.asyncio import AsyncSession
from fastapi import Depends
import structlog
from app.core.database import get_db

log = structlog.get_logger()
router = APIRouter(tags=["health"])

@router.get("/health/live")
async def liveness() -> dict:
    """Kubernetes liveness probe — is the process alive?"""
    return {"status": "alive"}

@router.get("/health/ready")
async def readiness(db: AsyncSession = Depends(get_db)) -> dict:
    """Kubernetes readiness probe — can we serve traffic?"""
    checks: dict[str, str] = {}
    overall = "ready"

    # DB check
    try:
        await db.execute(text("SELECT 1"))
        checks["db"] = "ok"
    except Exception as exc:
        checks["db"] = f"error: {exc}"
        overall = "degraded"
        log.error("health.db_check_failed", error=str(exc))

    # Redis check
    try:
        from app.core.redis import get_redis
        redis = get_redis()
        if redis:
            await redis.ping()
            checks["redis"] = "ok"
        else:
            checks["redis"] = "not_configured"
    except Exception as exc:
        checks["redis"] = f"error: {exc}"
        overall = "degraded"
        log.error("health.redis_check_failed", error=str(exc))

    status_code = 200 if overall == "ready" else 503
    from fastapi.responses import JSONResponse
    return JSONResponse(
        content={"status": overall, **checks},
        status_code=status_code,
    )
```

```markdown
# Backup & Restore Runbook

## PostgreSQL

### Manual backup
```bash
docker compose exec db pg_dump -U $POSTGRES_USER $POSTGRES_DB > backup-$(date +%Y%m%d).sql
```

### Restore
```bash
docker compose exec -T db psql -U $POSTGRES_USER $POSTGRES_DB < backup-YYYYMMDD.sql
```

### Automated backups
Set `PG_BACKUP_SCHEDULE` in your hosting environment. Recommended: daily at 02:00 UTC, retain 7 days.

## Redis

Redis data (conversation history, sessions) is ephemeral by design. No backup needed.
Session loss = users re-login. Conversation history TTL is 1h — acceptable data loss.

## Alembic migrations

Every schema change has a migration in `backend/alembic/versions/`.
To verify the DB is at the expected revision:
```bash
docker compose exec backend uv run alembic current
docker compose exec backend uv run alembic history
```

## Disaster recovery checklist

1. Restore DB from latest backup
2. `docker compose exec backend uv run alembic upgrade head`
3. `docker compose restart backend worker`
4. Verify: `curl http://localhost:8000/health/ready`
5. Smoke test: login, load a course, run one agent action

Recovery time objective: < 30 min with a recent backup.
```
